# Interview Questions

1. What is GitHub Pages?

GitHub Pages is a free hosting service from GitHub for static websites. It lets you publish HTML, CSS, and JavaScript directly from a repository.

2. Can you host dynamic applications on GitHub Pages?

No, GitHub Pages is intended for static sites. Dynamic applications usually require a backend server or external hosting platform.

3. What are the limitations of GitHub Pages?

It does not support server-side code, databases, or full backend applications. It is best suited for static content and simple frontend projects.

4. How do you update a GitHub Pages website?

You update the website by pushing changes to the repository branch connected to GitHub Pages. The site then rebuilds and publishes automatically.

5. What happens if the repository is deleted?

The GitHub Pages site will no longer be available because it is hosted from that repository.

6. What is the default file loaded by GitHub Pages?

The default page is usually index.html when it exists in the published branch.

7. Can you use a custom domain with GitHub Pages?

Yes, GitHub Pages supports custom domains when configured properly in the repository settings and DNS.
